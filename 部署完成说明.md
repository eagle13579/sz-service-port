## 🔍 常见问题

### Q1: 如何修改网站内容？

**A:** 直接编辑 `index.html` 文件，然后推送到 GitHub。

```powershell
# 编辑文件
notepad index.html

# 保存后提交
git add index.html
git commit -m "更新网站内容"
git push
```

### Q2: 网站更新后多久生效？

**A:** 通常 1-3 分钟。您可以在 Actions 页面查看部署状态。

### Q3: 如何查看部署日志？

**A:** 访问 Actions 页面，点击具体的部署记录即可查看详细日志。

### Q4: 推送时提示连接失败怎么办？

**A:** 可能是网络问题，尝试以下方法：

1. 多试几次：`git push`
2. 测试连接：`Test-NetConnection github.com -Port 443`
3. 检查网络连接
4. 如果使用代理，配置 Git 代理：
   ```powershell
   git config --global http.proxy http://127.0.0.1:你的代理端口
   git config --global https.proxy http://127.0.0.1:你的代理端口
   ```

### Q5: 如何回退到之前的版本？

**A:**
```powershell
# 查看历史版本
git log

# 回退到指定版本
git reset --hard <commit-hash>

# 强制推送
git push -f origin main
```

### Q6: 可以使用私有仓库吗？

**A:** GitHub Pages 免费版本只支持公开仓库。私有仓库需要使用 GitHub Pro 付费版。

### Q7: 如何查看网站访问量？

**A:** GitHub Pages 本身不提供访问统计。您可以：
1. 集成第三方统计工具（如 Google Analytics）
2. 使用 Cloudflare Web Analytics
3. 使用 Umami 等开源统计工具
