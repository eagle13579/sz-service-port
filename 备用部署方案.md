# 数智服务港 - 备用部署方案

由于云开发默认域名过期，提供以下备用部署方案：

---

## 方案一：本地快速启动（推荐用于演示）

### Windows用户

1. **双击运行启动脚本**
   ```
   start-system.bat
   ```

2. **浏览器访问**
   ```
   http://localhost:5173
   ```

### Linux/Mac用户

```bash
chmod +x start-system.sh
./start-system.sh

# 浏览器访问
open http://localhost:5173
```

## 方案二：Vercel部署（推荐用于生产）

Vercel提供免费、稳定的部署服务。

### 步骤

1. **安装Vercel CLI**
```bash
npm install -g vercel
```

2. **登录Vercel**
```bash
vercel login
```

3. **部署前端**
```bash
cd unified-frontend
npm install
npm run build
vercel --prod
```

4. **获取访问地址**
   部署完成后，Vercel会提供一个稳定的URL，例如：
   ```
   https://shuzhi-hub-frontend.vercel.app
   ```

### 优点
- ✅ 完全免费
- ✅ 全球CDN加速
- ✅ HTTPS自动配置
- ✅ 自定义域名支持
- ✅ 不会过期

---

## 方案三：Netlify部署

Netlify是另一个优秀的静态网站托管平台。

### 步骤

1. **安装Netlify CLI**
```bash
npm install -g netlify-cli
```

2. **登录Netlify**
```bash
netlify login
```

3. **部署**
```bash
cd unified-frontend
npm install
npm run build
netlify deploy --prod --dir=dist
```

4. **获取访问地址**
   部署完成后会提供URL，例如：
   ```
   https://shuzhi-hub.netlify.app
   ```

### 优点
- ✅ 免费额度大
- ✅ 功能丰富
- ✅ 支持表单、函数等
- ✅ 不会过期

## 方案四：GitHub Pages

如果代码已在GitHub上，可以使用GitHub Pages。

### 步骤

1. **推送代码到GitHub**
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/shuzhi-hub.git
git push -u origin main
```

2. **配置GitHub Pages**
   - 访问仓库 → Settings → Pages
   - Source: Deploy from a branch
   - Branch: main / (root)
   - 点击Save

3. **等待部署**
   通常需要1-2分钟，部署完成后访问：
   ```
   https://your-username.github.io/shuzhi-hub/
   ```

### 优点
- ✅ 完全免费
- ✅ 与GitHub集成
- ✅ 稳定可靠
- ✅ 支持自定义域名

## 方案五：本地Web服务器

如果只是临时查看，可以使用本地Web服务器。

### 使用Python（推荐）

```bash

cd unified-frontend
python -m http.server 8000

### 使用Node.js http-server

# http://localhost:8000
```

### 使用PHP

```bash
cd unified-frontend
php -S localhost:8000

# 访问
# http://localhost:8000
```

## 方案六：云服务器部署

如果有自己的云服务器，可以部署到服务器上。

### 使用Nginx

1. **构建前端**
```bash
cd unified-frontend
npm install
npm run build
```

2. **上传到服务器**
```bash
scp -r dist/* user@your-server:/var/www/html/
```

3. **配置Nginx**
```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

4. **重启Nginx**
```bash
sudo systemctl restart nginx
```

### 优点
- ✅ 完全控制
- ✅ 可以配置后端服务
- ✅ 性能优秀

## 快速决策指南

### 我想要立即看到效果
→ **使用本地启动**
```bash
start-system.bat  # Windows
./start-system.sh  # Linux/Mac
```

### 我想要一个稳定的在线地址
→ **使用Vercel**
```bash
npm install -g vercel
vercel login
cd unified-frontend && vercel --prod
```

### 我想要完全免费且稳定
→ **使用GitHub Pages**
1. 推送到GitHub
2. 开启GitHub Pages

### 我有自定义域名
→ **使用Netlify或Vercel**
1. 部署到平台
2. 绑定自定义域名

### 我想要部署完整的后端+前端
→ **使用云服务器**
1. 部署后端Node.js服务
2. 部署前端静态文件
3. 配置Nginx反向代理
